Explain Von Neumann Architecture.
ARCH
▾

Von Neumann architecture is a computer design where instructions and data are stored in the same memory space. The CPU fetches both from the same memory using the same pathways.

Types of Computers

  • Fixed Program Computers — Function-specific, cannot be reprogrammed (e.g., calculators).
  • Stored Program Computers — Programmable; applications are stored in memory.

3 Main Components

1. CPU (Central Processing Unit)

The CPU is the brain of the computer. It consists of the Control Unit, ALU, and Registers.

  • CU (Control Unit) — Sends control signals; manages data flow, I/O, and instruction fetch.
  • ALU (Arithmetic Logic Unit) — Handles arithmetic (add, sub) and logical (comparisons, shifts) operations.

Registers (Fastest Memory, Inside CPU)

RegisterFull NameFunction
ACAccumulatorStores ALU results
PCProgram CounterHolds address of next instruction
MARMemory Address RegisterStores memory address to access
MDRMemory Data RegisterStores data fetched/to be stored
IR (CIR)Instruction RegisterHolds current instruction being decoded
GPRGeneral Purpose RegistersTemp data storage during processing

2. Memory

Stores both data and instructions (the defining feature of Von Neumann design).

3. I/O Devices

Input devices send data to main memory; output devices present results to the user.

Buses

  • Control Bus — Carries control signals and status flags.
  • Address Bus — Carries memory addresses (not data).
  • Data Bus — Carries actual data between CPU, memory, and I/O.
Key Feature: Von Neumann Bottleneck — since data and instructions share the same bus, they cannot be fetched simultaneously, limiting performance.

Define: 1. Multiprocessor   2. Multicomputer
ARCH
▾

1. Multiprocessor

A system with two or more CPUs that share common resources.

  • Common main memory
  • Common clock
  • Shared I/O devices
  • Single operating system
Think of it as multiple workers sharing one office with one manager (OS).

2. Multicomputer

Multiple independent computers (nodes) connected via a communication network.

  • Each node has its own private memory
  • Each node has its own operating system
  • Each node has its own processor
Think of it as separate offices in different buildings, communicating over a network.

Explain Floating Point and Fixed Point Representation.
NUMBER SYS
▾

Fixed Point Representation

The binary point is always at a fixed position. Two common positions:

  • Extreme left → number is a fraction
  • Extreme right → number is an integer

Integer (Signed) Representation

Positive numbers: sign bit = 0, magnitude follows.

Negative numbers: sign bit = 1, rest represented in one of three ways:

Method-14 in 8 bitsHow Obtained
Signed Magnitude1 0001110Complement only the sign bit of +14
Signed 1's Complement1 1110001Complement ALL bits of +14
Signed 2's Complement1 11100101's complement + 1
Most CPUs use 2's complement for subtraction — only an adder is needed, reducing hardware complexity.

Floating Point Representation (IEEE 754)

Numbers stored as {Sign, Mantissa, Exponent}. The radix point is not fixed.

(sign) × (mantissa) × 2^(±exponent)
PrecisionTotal BitsSignExponentMantissaBias
Single32 bits1823127
Double64 bits111521023

Steps to Represent a Number in Floating Point

  1. Convert number to signed binary
  2. Normalize: move radix point after first 1 → get exponent
  3. Add bias (127 for 32-bit, 1023 for 64-bit) to exponent
  4. Convert biased exponent to binary
  5. Store mantissa bits (bits after decimal point, padded to 23/52 bits)
  6. Set sign bit (0 = positive, 1 = negative)

Example: 110.111₂

110.111 = 1.10111 × 2² (shifted 2 places → exponent = 2) Biased exp = 2 + 127 = 129 = 10000001₂ Sign: 0 Format: 0 | 10000001 | 10111000000000000000000

A program in memory is executed by going through a cycle for each instruction. Each cycle is subdivided into phases/sub-cycles.

Basic Instruction Cycle Phases

┌──────────────────────┐ │ START │ └──────────┬───────────┘ ▼ ┌──────────────────────┐ │ 1. FETCH │ ← Get instruction from memory (PC → MAR → IR) └──────────┬───────────┘ ▼ ┌──────────────────────┐ │ 2. With Interrupt Handling (Extended Cycle)

  1. Fetch instruction from memory
  2. Decode the instruction
  3. Execute the instruction
  4. Check for interrupts — if yes, save state and process interrupt; then return
The cycle repeats indefinitely until a HALT instruction is encountered.

Signed representation of integer. Write (–12)₁₀ in 8-bit Signed Magnitude, 1's Complement, 2's Complement.
NUMBER SYS
▾

In binary, the MSB (leftmost bit) is the sign bit: 0 = positive, 1 = negative.

Representations of negative numbers

  1. Signed Magnitude — Only sign bit is complemented
  2. Signed 1's Complement — All bits (including sign) are complemented
  3. Signed 2's Complement — 1's complement + 1

Example: Represent (–12)₁₀ in 8 bits

First, +12 in 8-bit binary = 0 0001100

+12 = 0 0001100 Signed Magnitude (-12): → Flip only sign bit → 1 0001100 Signed 1's Complement (-12): → Flip ALL bits → 1 1110011 Signed 2's Complement (-12): → 1's complement of +12: 1 1110011 → Add 1: 1 1110011 + 1 = 1 1110100
Method-12 in 8 bits
Signed Magnitude1 0001100
Signed 1's Complement1 1110011
Signed 2's Complement1 1110100
Most computers use 2's complement because: no separate subtractor needed (add the 2's complement = subtract), and there is only one representation of zero.

RTL — Register Transfer Language

A symbolic language used to describe micro-operations performed between registers in a digital system.

RTL specifies: Source register, Destination register, and Control condition.

Notation

P : R2 ← R1

Meaning: When control signal P = 1, contents of R1 are transferred to R2.

Discuss 4-bit Binary Adder with neat diagram.
ALU
▾

Structure

A 4-bit binary adder uses 4 Full Adders (FA) connected in a chain (ripple carry).

B3 A3 B2 A2 B1 A1 B0 A0 ↓ ↓ ↓ ↓ ↓ ↓ ↓ ↓ ┌────┐ ┌────┐ ┌────┐ ┌────┐ │ FA │←C│ FA │←C│ FA │←C│ FA │← C0 (input carry) └─┬──┘ └─┬──┘ └─┬──┘ └─┬──┘ ↓ ↓ ↓ ↓ C4=Cout S3 S2 S1 S0

Key Points

  • Input carry to the adder is C0; output carry is C4
  • The S outputs (S0–S3) give the sum bits
  • An n-bit adder requires n full adders
  • Output carry of each FA connects to input carry of the next higher FA (ripple carry)
  • A inputs come from register R1, B inputs from register R2
  • Sum can be stored back to R1, R2, or a third register
Ripple Carry Adder: carry must propagate through all FAs → delay increases with n bits. Solution: Carry Lookahead Adder (faster).

Write assembly language code in GNUsim8085 to subtract two 8-bit numbers.
ASSEMBLY
▾

Problem

Subtract two 8-bit numbers stored at memory locations 2000H and 2001H. Store result at 2002H.

Algorithm

  1. Load first number into A
  2. Move A to B (save first number)
  3. Load second number into A
  4. Subtract B from A → A = A - B
  5. Store result
  6. Halt
LDA 2000H ; A ← Memory[2000H] (load first number) MOV B, A ; B ← A (save first number in B) LDA 2001H ; A ← Memory[2001H] (load second number) SUB B ; A ← A - B (subtract) STA 2002H ; Memory[2002H] ← A (store result) HLT ; Stop execution
SUB B internally uses 2's complement: A = A + (2's complement of B). Result and flags (carry, zero, sign) are updated in the flag register.

Explain any two Shift Micro-operations with examples.
ALU
▾

Shift micro-operations move bits in a register left or right. Three types:

1. Logical Shift

Shifts bits left or right; fills vacated bits with 0.

Logical Shift Left (shl): Register: 1 0 1 1 0 1 0 0 After: 0 1 1 0 1 0 0 0 ← 0 fills from right RTL: R ← shl R (serial input = 0) Logical Shift Right (shr): Register: 1 0 1 1 0 1 0 0 After: 0 1 0 1 1 0 1 0 ← 0 fills from left RTL: R ← shr R (serial input = 0)

2. Circular Shift (Rotate)

Bits shifted out from one end come back in from the other end. No bits lost.

Circular Shift Left (cil): Register: 1 0 1 1 0 1 0 0 After: 0 1 1 0 1 0 0 1 ← MSB wraps to LSB RTL: R ← cil R Circular Shift Right (cir): Register: 1 0 1 1 0 1 0 0 After: 0 1 0 1 1 0 1 0 → wait, LSB (0) wraps to MSB RTL: R ← cir R

3. Arithmetic Shift

Preserves the sign bit. Used for multiplication/division by 2.

Arithmetic Shift Right: Register: 1 0 1 1 0 1 0 0 After: 1 1 0 1 1 0 1 0 ← sign bit (1) replicated (÷ 2 while preserving sign)
Arithmetic shift left by 1 = multiply by 2. Arithmetic shift right by 1 = divide by 2.

Explain Arithmetic Micro-operations.
ALU
▾

Arithmetic micro-operations perform basic numerical computations on data stored in registers.

OperationRTL NotationDescription
AddR3 ← R1 + R2Adds contents of R1 and R2
SubtractR3 ← R1 − R2Subtracts R2 from R1
IncrementR1 ← R1 + 1Adds 1 to register
DecrementR1 ← R1 − 1Subtracts 1 from register
Negate (2's comp)R2 ← R̄1 + 12's complement (arithmetic negation)
Add with CarryR3 ← R1 + R2 + CinAdds with carry-in bit
Subtract with BorrowR3 ← R1 − R2 − BinSubtracts with borrow bit

Hardware Implementation

A single Binary Adder can implement all the above by clever input manipulation:

  • To subtract: apply 2's complement of R2 (invert bits + set Cin = 1)
  • To increment: set B input = 0 and Cin = 1
  • To decrement: set B = all 1s (= −1 in 2's complement) and Cin = 0
This is why modern CPUs only need an adder — not separate hardware for each operation. 2's complement unifies addition and subtraction.

Explain Selective Set, Selective Complement, and Selective Clear operations.
ALU
▾

These are logic micro-operations that selectively modify specific bits in register A using register B as a mask.

1. Selective Set (OR)

Sets bits in A to 1 where B has 1. Doesn't affect bits where B = 0.

A ← A ∨ B (OR operation)
A: 0 1 0 1 1 0 0 0 B: 0 0 0 0 0 1 1 1 A∨B: 0 1 0 1 1 1 1 1 ← last 3 bits SET to 1

2. Selective Complement (XOR)

Complements bits in A where B has 1. Doesn't affect bits where B = 0.

A ← A ⊕ B (XOR operation)
A: 0 1 0 1 1 0 0 0 B: 0 0 0 0 0 1 1 1 A⊕B: 0 1 0 1 1 1 1 1 ← last 3 bits COMPLEMENTED (0→1)

3. Selective Clear (AND with complement)

Clears bits in A to 0 where B has 1. Doesn't affect bits where B = 0.

A ← A ∧ B' (AND with NOT B)
A: 0 1 0 1 1 0 0 0 B: 0 0 0 0 0 1 1 1 B': 1 1 1 1 1 0 0 0 A∧B': 0 1 0 1 1 0 0 0 ← last 3 bits CLEARED to 0
OperationRTLEffect where B=1Effect where B=0
Selective SetA ← A ∨ BSet to 1Unchanged
Selective ComplementA ← A ⊕ BToggleUnchanged
Selective ClearA ← A ∧ B'Clear to 0Unchanged
B acts as a mask — it selects WHICH bits of A to operate on. This is fundamental to bit manipulation in low-level programming.
